# 同步与备份

笔记是资产，备份不可省。以下是主流方案。

## 方案对比

| 方案 | 费用 | 跨平台 | 版本历史 | 难度 |
|------|------|--------|----------|------|
| **Obsidian Sync** | $5/月 | 全平台 | 有 | 零 |
| **Git** | 免费 | 电脑端 | 完整 | 中 |
| **iCloud** | 免费/5GB | Apple 生态 | 有限 | 低 |
| **OneDrive/Google Drive** | 免费 | 全平台 | 有限 | 低 |

## Obsidian Sync（推荐）

官方同步方案，端到端加密。

- `设置 → Obsidian Sync` 登录并开启
- 自动同步所有笔记、设置、插件、主题
- 可按文件夹选择性同步
- 支持版本历史回退

> 优点：零配置、省心、安全。缺点：付费。

## Git 同步

就是你现在用的方式。

```
git add -A
git commit -m "update"
git push
```

- 完全掌控，完整的变更历史
- 推荐搭配 Obsidian Git 社区插件实现自动提交
- 不适用于 iOS（需第三方 Git 客户端如 Working Copy）

## iCloud / OneDrive

把仓库文件夹放入云盘目录即可。

- **iCloud**: Mac + iPhone/iPad 无缝，Windows 端体验一般
- **OneDrive**: Windows 好，Mac 可接受，但大量小文件同步可能卡顿

> 注意：**不要同时使用多个同步方案**，容易产生冲突文件。

## .gitignore 建议

如果用 Git，排除频繁变化的工作区文件：
```
.obsidian/workspace.json
.obsidian/workspace-mobile.json
.trash/
```

## 最佳实践

- 至少保留一种自动备份方式
- 网络盘 ≠ 备份（误删会同步删除），Git 或 Sync 的版本历史才是真备份
- 定期导出完整仓库为 ZIP 存档
