# Properties 与元数据

Properties（属性）是笔记顶部的 YAML 元数据，是 Obsidian 数据结构化的基础。

## 添加属性

- `Ctrl+;` 打开属性面板
- 或笔记顶部输入 `---` 进入 YAML 编辑模式

## 属性类型

| 类型 | 示例 | 说明 |
|------|------|------|
| 文本 | `author: 鲁迅` | 普通文字 |
| 数字 | `rating: 4.5` | 可用于数值比较 |
| 复选框 | `finished: true` | 是/否 |
| 日期 | `due: 2026-06-01` | 日历选择器 |
| 日期时间 | `meeting: 2026-06-01T14:00` | 含时间 |
| 列表 | `tags: [读书, 文学]` | 多值 |
| 链接 | `project: "[[项目A]]"` | 笔记链接 |

## 内置属性

Obsidian 自动维护的属性，无需手动编辑：

| 属性 | 说明 |
|------|------|
| `tags` | 标签列表 |
| `aliases` | 别名（搜索可用别名匹配） |
| `cssclasses` | 为单篇笔记加载特定 CSS |

## 实际用途

### 配合 Dataview

```
---
rating: 5
author: 村上春树
genre: [小说, 日本文学]
---
```

在 Dataview 中可查询、排序、过滤。

### 配合模板

```
---
tags: [日记]
created: {{date}}
mood: 
---
```

Templater 可读取属性值做条件处理。

### 配合搜索

搜索 `[rating:5]` 可精确找到所有五星内容。

## 最佳实践

- 属性不宜过多，5-8 个足够
- 同类笔记保持属性一致（如所有读书笔记统一用 `author`、`rating`）
- 用 `aliases` 给笔记取别名，搜索更灵活
